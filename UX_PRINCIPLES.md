# UX Principles

## 유효성 검증 실패 처리

- **alert/팝업 금지** — 흐름을 끊지 않는다
- 문제 요소를 **흔들기(animate-shake)** 로 강조
- 해당 요소 바로 아래에 **빨간 안내문구(text-danger-fg, text-xs)** 표시
- 유저가 수정하기 전까지 문구는 사라지지 않음
- 단, 유저가 아직 시도하지 않은 상태에서는 미리 노출하지 않음 — 실패 액션 이후에만 표시

## CTA 아래 이탈 경로 버튼

- `EscapeButton` 컴포넌트 사용 (`src/components/ui/escape-button.tsx`)
- 로그인, 약관 동의, 프로필 입력 등 온보딩 플로우 전반에 적용

## 약관 동의

- 필수 항목은 반드시 개별 체크박스로 명시적 동의 수집 (개인정보보호법 제22조)
- "전체 동의" 는 편의 기능으로만 제공, 개별 항목 체크박스와 병행
- 미동의 상태에서 CTA 클릭 시: 체크박스 흔들기 + 인라인 안내문구 표시

## 레이아웃

- 온보딩 페이지의 CTA는 하단 고정(fixed bottom) 배치
- 콘텐츠는 상단 정렬, 하단 고정 CTA에 가려지지 않도록 pb 여백 확보
